# 评分与热度说明

有希计划的评分和热度数据来自多个主流平台（Bangumi、MyAnimeList、AniList），经过归一化处理后综合计算得出。本文介绍其背后的计算方式。

---

## 评分

### 第一步：归一化

不同平台的打分习惯差异显著——同样是 8 分，在 Bangumi 上算优秀，在 MAL 上可能只是平庸。为消除平台间的尺度差异，每部番剧的原始评分首先会被转换为 **Z-score**：

$$
z = \frac{score - \mu}{\sigma}
$$

- **score**：该番剧在此平台的原始评分
- **μ**：该平台所有番剧的平均分
- **σ**：该平台评分的标准差

Z-score 表达的是“这部番剧比平台平均水平高（或低）了多少个标准差”，从而将不同平台的评分拉到同一把尺子上衡量。

### 第二步：映射到百分制

Z-score 以 **±3σ** 为参考区间，线性映射到更直观的 0–100 分：

$$
normalizedScore = \frac{z + 3}{6} \times 100
$$

| z 值 | 对应分数 | 含义           |
| ---- | -------- | -------------- |
| −3   | 0 分     | 平台内垫底水平 |
| 0    | 50 分    | 平台平均水平   |
| +3   | 100 分   | 平台顶尖水平   |

> 极少数评分极端的番剧（z 超出 ±3 范围）最终分数可能低于 0 或高于 100，属于正常现象。

### 第三步：加权平均

各平台归一化后的分数，按各自权重加权平均，得出最终综合评分：

$$
\text{Final Score} = \frac{\sum (normalizedScore_i \times weight_i)}{\sum weight_i}
$$

权重反映了各平台数据的可靠性与参考价值，权重越高的平台对最终结果影响越大。

> 若某个平台对该番剧无评分，则该平台不参与计算。
>
> 若所有平台均无评分数据，综合评分显示为空。

## 热度

热度反映一部番剧在各平台上的关注人数总和，数值越高说明受众越广。

### 第一步：归一化

各平台的用户规模差异悬殊（MAL 动辄数十万关注，Bangumi 通常只有数千），直接相加会让 MAL 几乎完全主导结果。为此，各平台的原始关注人数先乘以对应**倍率**进行换算：

$$
normalizedPopularity = popularity \times multiplier
$$

- **popularity**：该番剧在此平台的原始关注人数
- **multiplier**：该平台的热度倍率，以 MAL 为基准（倍率 1.0）

> 各平台的数据来源略有不同：AniList 直接提供专门的 popularity 字段，而 Bangumi 和 MAL 则以**评分人数**作为热度的代理指标。

> 倍率通过数据统计得出：对所有同时在 MAL 与目标平台有记录的番剧，计算两者关注人数的比值，取**中位数**作为最终倍率。

### 第二步：累加

$$
\text{Total Popularity} = \sum normalizedPopularity_i
$$

与评分不同，热度直接累加各平台的归一化值，覆盖平台越多、关注人数越多，热度值越高。

> 若某个平台对该番剧无关注数据，则跳过该平台。

---

各平台的具体权重、评分均值、标准差及热度倍率，参见[平台数据参数](/docs/platforms)。
